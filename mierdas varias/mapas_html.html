Mapas en HTML — <map>, SVG y embeds (Todo comentado)

Mapa de imagen (<img usemap> + <map>)

Ejemplo con una imagen ficticia (usa tus propias coordenadas en función de la imagen real).

Consejos: las coordenadas deben calcularse con la resolución original de la imagen. Para imágenes responsive puedes usar scripts o imágenes SVG (ver más abajo) o usar atributos srcset/picture y mapster en JS si fuese necesario — aquí pediste sin JS, así que recomendamos SVG para responsividad pura.

Mapa interactivo con SVG (responsive, sin JavaScript)

SVG permite formas precisas, escalables y accesibles. Aquí cada <rect> o <circle> está envuelto en <a> para enlaces.

Ventaja: SVG escala sin pérdida y mantiene las áreas clicables correctamente en cualquier tamaño.

Mapa embebido (iframe) — OpenStreetMap

Usa un iframe si quieres mostrar un mapa real (OpenStreetMap, Google Maps, etc.). No requiere JS en tu HTML; el servicio inserta su propio JS dentro del iframe.

Consejo: para Google Maps necesitarás una API key si quieres usar la API; el embed iframe estándar no siempre requiere clave. Revisa las políticas del proveedor antes de usar en producción.

Imagen + SVG overlay (coords responsive sin JS)

Coloca la imagen dentro de un contenedor y superpone un SVG en la misma vista usando viewBox proporcional.

Recomendado para planos responsive cuando no se quiere usar JS. Asegúrate que la relación de aspecto de la imagen y del viewBox coincidan.

Accesibilidad y alternativas

Siempre ofrece alternativas: una lista de enlaces debajo del mapa para quienes no puedan interactuar con el gráfico.

  • Ir a Zona 1 (enlace alternativo)
  • Ir a Zona 2
  • Ir a Zona 3

También incluye descripciones textuales (captions) y asegúrate de que los elementos sean alcanzables por teclado (enlaces <a> son focusables por defecto).
