1) Select nativo (<select>)

El control nativo es el más accesible y funciona en todos los navegadores y dispositivos. Ideal para formularios.

2) <details> / <summary> (acordeón nativo)

Elemento semántico que permite mostrar/ocultar contenido con click sin JS. Soporta teclado y es accesible.

Notas de accesibilidad
  • Usa <label> para los controles de formulario (select, checkbox).
  • Preferir el <select> nativo para formularios donde el usuario espera un control de selección.
  • Si usas dropdown personalizados, añade roles (menu, menuitem) y asegúrate de la navegación por teclado si implementas con JS; las soluciones sin JS tienen limitaciones en accesibilidad de teclado.
  • En móviles, :hover no funciona; considera usar la técnica del checkbox o <details> para click.
